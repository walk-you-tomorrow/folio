# Folio — 브랜드 가이드라인 v1.0

> 작성일: 2026-04-07

---

## 1. 브랜드 개요

**서비스명:** Folio (폴리오)

**한 문장 정의:**
회원가입 없이 링크 하나로 입장하여, 무한 캔버스 위에 텍스트·그림·사진을 자유롭게 올리고 실시간으로 함께 보는 즉석 협업 메모장

**핵심 컨셉:**
모든 사람들의 생각이 하나의 캔버스로 무한하게 확장해 나간다

**태그라인:**
"Your thoughts, unfolded together."

---

## 2. 미션 & 비전

**미션:**
누구든 지금 이 순간, 생각을 꺼내 함께 놓을 수 있는 공간을 만든다.

**비전:**
세상의 모든 협업이 로그인 없이, 준비 없이, 지금 바로 시작되는 것.

---

## 3. 브랜드 퍼스널리티

| 키워드 | 설명 |
|---|---|
| **즉각적(Instant)** | 준비할 것도, 기다릴 것도 없다. 생각이 생긴 그 순간 시작된다. |
| **가벼운(Light)** | 부담 없이 쓰고, 부담 없이 공유하고, 부담 없이 떠나도 된다. |
| **개방적(Open)** | 누구에게나 열려 있고, 모든 아이디어를 환영한다. |
| **활기찬(Alive)** | 정적인 문서가 아니라 살아 움직이는 생각의 공간. |
| **솔직한(Honest)** | 과장하지 않고, 필요한 것만 한다. 군더더기 없이 본질에 집중. |

## 4. 톤 앤 매너

**기본 방향:**
- 격식 없는 해요체. 지나치게 캐주얼하거나 유행어 남발 금지.
- 기능 설명보다 사용자가 얻는 경험 중심으로 문장 구성.
- 짧고 선명하게. 한 문장에 하나의 메시지.

### DO / DON'T

| 상황 | DO | DON'T |
|---|---|---|
| 서비스 소개 | "링크 하나로 바로 시작하세요." | "혁신적인 실시간 협업 솔루션을 경험해보십시오." |
| 에러 발생 | "잠깐 멈췄어요. 다시 시도해볼게요." | "오류가 발생했습니다. 관리자에게 문의하세요." |
| 기능 유도 | "스티커를 끌어다 놓아보세요." | "텍스트 스티커 기능을 활용하시기 바랍니다." |
| 공유 유도 | "이 링크만 보내면 돼요." | "초대 링크를 생성하여 협업자에게 전달하세요." |
| 빈 캔버스 | "여기서부터 시작하면 돼요." | "캔버스가 비어 있습니다. 콘텐츠를 추가하세요." |

## 5. 비주얼 디렉션

### 5.1 컬러 팔레트

| 역할 | 이름 | Hex | 사용 맥락 |
|---|---|---|---|
| **Primary** | Folio Blue | `#3B6FFF` | 주요 버튼, 강조 링크, 핵심 UI 요소 |
| **Secondary** | Soft Indigo | `#7C5CFC` | 보조 강조, 호버 상태, 일러스트 포인트 |
| **Accent** | Lemon Pop | `#FFD43B` | 알림 뱃지, 하이라이트, 긍정 피드백 |
| **Background** | Canvas White | `#F7F7F5` | 캔버스 기본 배경, 전체 페이지 배경 |
| **Surface** | Cool Gray | `#EEEEF0` | 패널, 카드, 툴바 배경 |
| **Text Primary** | Deep Ink | `#1A1A2E` | 제목, 본문 텍스트 |
| **Text Secondary** | Quiet Gray | `#6B6B80` | 설명 텍스트, 플레이스홀더 |
| **Success** | Mint Go | `#22C55E` | 저장 완료, 연결 성공 |
| **Error** | Soft Red | `#F87171` | 에러, 경고 (공격적이지 않게) |

### 5.2 타이포그래피

**한국어:** Pretendard (Regular, Medium, SemiBold, Bold)
**영문:** Plus Jakarta Sans (제목) / Inter (본문)

| 용도 | 크기 | 굵기 | 비고 |
|---|---|---|---|
| 제목 | 28-40px | Bold/ExtraBold | letter-spacing -0.5px |
| 본문 | 15-17px | Regular/Medium | line-height 1.6 |
| UI 레이블 | 13-14px | SemiBold | 대문자 사용 지양 |

### 5.3 아이콘/일러스트 스타일

- 선 굵기 1.5-2px 아웃라인 기반, Rounded Corner 2-4px
- Primary Blue, Accent Yellow를 부분적으로 채워 사용
- 기하학적 도형 + 손 그린 듯한 곡선 혼합 ("스케치한 느낌")
- 과도한 3D, 사진 합성 금지. Flat하되 무표정하지 않게

### 5.4 UI 원칙

- 모서리 반경: 버튼 8px, 카드/패널 12px, 스티커 6px
- 그림자: 최소화. `0 2px 8px rgba(0,0,0,0.08)` 이하
- 여백: 넉넉하게. 4px 베이스 그리드
- Primary CTA 버튼: 한 화면에 1개만
- 컬러 변수: CSS Custom Properties로 관리

## 8. 팀별 브랜드 적용 가이드

### 디자이너
- 캔버스가 주인공. UI는 캔버스를 지지하는 틀.
- 팔레트 정의된 색상만 사용. 임의 컬러 추가 시 브랜드 매니저 협의 필수.
- Primary CTA는 Folio Blue(`#3B6FFF`) 단독. 같은 화면에 2개 이상 금지.
- 폰트는 Pretendard 단일 패밀리.
- 아이콘: Rounded Outline 통일. Filled는 active 상태에만.

### 개발자
- "즉각적" 브랜드 = 성능이 브랜드. 캔버스 첫 렌더 3초 이내.
- 실시간 동기화 500ms 이상 지연 시 상태 메시지 표시.
- 에러 메시지에 개발자 코드 노출 금지. 브랜드 마이크로카피로 대체.
- 컬러는 CSS Custom Properties로 관리.
- font-display: swap 및 프리로드 적용.
- 접근성: aria-label, 키보드 네비게이션 지원.

### 마케터
- 설득하지 말고 경험하게 하라.
- 기능 나열 대신 사용 시나리오 보여주기.
- "혁신적인", "최고의", "업계 최초" 등 과장 수식어 금지.
- 경쟁사 직접 비교 광고 금지.
- CTA: "지금 바로 시작하기" 또는 "링크 만들어보기".

### CS
- 해요체 사용. 격식체/구어체 금지.
- 사과는 1번, 간결하게. 해결 방법 먼저 안내.
- 기술 용어는 사용자 언어로 번역.
- 고객 탓 표현 금지.
