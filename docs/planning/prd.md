> 작성일: 2026-04-07

## 1. 서비스 개요

**서비스명:** Folio (폴리오)
**한 문장 정의:** 회원가입 없이 링크 하나로 입장하여, 무한 캔버스 위에 텍스트·그림·사진을 자유롭게 올리고 실시간으로 함께 보는 즉석 협업 메모장
**태그라인:** "Your thoughts, unfolded together."

### 목표
- 아이디어 공유까지의 시간을 **30초 이내**로 단축
- 회원가입/설치 없이 **링크 클릭 한 번**으로 협업 시작
- MVP 7일 내 론치, 초기 사용자 피드백 수집

## 2. 타겟 사용자

### 페르소나 A: "회의하는 지은" (직장인, 29세)
- **상황:** 팀 브레인스토밍에서 아이디어를 모으고 싶지만 Miro 계정 만들기는 번거로움
- **니즈:** 지금 바로, 준비 없이, 다 같이 적을 수 있는 공간
- **사용 시나리오:** 회의 시작 → Folio 링크 생성 → 슬랙에 공유 → 각자 스티커로 아이디어 붙이기

### 페르소나 B: "과제하는 민수" (대학생, 22세)
- **상황:** 조별과제 아이디어를 카톡으로 주고받는데 정리가 안 됨
- **니즈:** 사진, 텍스트를 한 곳에 모아서 같이 볼 수 있는 공간
- **사용 시나리오:** 카톡에 Folio 링크 공유 → 참고 이미지 업로드 → 의견 스티커 붙이기

### 페르소나 C: "기획하는 소영" (프리랜서 디자이너, 34세)
- **상황:** 클라이언트에게 무드보드를 빠르게 공유하고 피드백 받고 싶음
- **니즈:** 계정 없이 누구나 바로 볼 수 있는 비주얼 보드
- **사용 시나리오:** 레퍼런스 이미지 배치 → 링크 전달 → 클라이언트가 코멘트 스티커 추가

## 3. MVP 기능 요구사항

### 3.1 캔버스 생성 & 링크 공유

| 항목 | 스펙 |
|---|---|
| 생성 방식 | 홈 → "새 캔버스 만들기" 클릭 → 즉시 생성 |
| URL 형식 | `folio.app/{canvasId}` (nanoid 8자리) |
| 공유 방식 | URL 복사 버튼, 카카오톡/SNS 공유 버튼 |
| 접근 권한 | 링크를 가진 누구나 편집 가능 (MVP) |
| 캔버스 수명 | 생성 후 7일 (MVP), 이후 자동 삭제 |

**수용 기준:**
- [ ] 홈에서 1클릭으로 캔버스 생성 후 캔버스 화면 진입
- [ ] 고유 URL이 생성되어 브라우저 주소창에 표시
- [ ] "링크 복사" 클릭 시 클립보드에 복사 + 토스트 알림
- [ ] 링크로 접속한 사용자가 같은 캔버스에 진입

### 3.2 텍스트 스티커

| 항목 | 스펙 |
|---|---|
| 생성 | 툴바 "텍스트" 버튼 클릭 또는 캔버스 더블클릭 |
| 편집 | 인라인 텍스트 편집 (클릭하여 수정) |
| 이동 | 드래그 앤 드롭 |
| 크기 | 텍스트 길이에 따라 자동 조절, 수동 리사이즈 가능 |
| 색상 | 6가지 프리셋 배경색 (브랜드 팔레트 기반) |
| 삭제 | 선택 후 Delete키 또는 삭제 버튼 |
| 최대 글자 | 500자 |

**수용 기준:**
- [ ] 캔버스 위에 텍스트 스티커 생성/편집/삭제 가능
- [ ] 드래그로 자유롭게 위치 이동
- [ ] 다른 참여자에게 실시간 반영
- [ ] 배경색 변경 가능

### 3.3 이미지 업로드

| 항목 | 스펙 |
|---|---|
| 업로드 방식 | 툴바 버튼 클릭 / 캔버스에 드래그 앤 드롭 / 클립보드 붙여넣기 |
| 지원 포맷 | JPG, PNG, GIF, WebP |
| 최대 크기 | 5MB per file |
| 배치 | 업로드 위치 또는 캔버스 중앙에 배치 |
| 조작 | 드래그 이동, 모서리 핸들 리사이즈 |
| 삭제 | 선택 후 Delete키 또는 삭제 버튼 |

**수용 기준:**
- [ ] 3가지 방식(버튼/드래그/붙여넣기)으로 이미지 업로드
- [ ] 업로드 중 로딩 인디케이터 표시
- [ ] 이미지 드래그 이동 및 리사이즈
- [ ] 5MB 초과 시 에러 메시지 (브랜드 톤)

### 3.4 실시간 동기화

| 항목 | 스펙 |
|---|---|
| 동기화 대상 | 스티커/이미지 생성, 수정, 삭제, 위치 이동 |
| 지연 목표 | 500ms 이내 |
| 충돌 처리 | Last-Write-Wins (MVP) |
| 접속자 표시 | 우측 상단에 현재 접속자 아바타/닉네임 |
| 연결 끊김 | 자동 재연결 + 상태 메시지 표시 |

**수용 기준:**
- [ ] 한 사용자의 변경이 다른 사용자에게 500ms 이내 반영
- [ ] 접속자 수/닉네임 실시간 표시
- [ ] 연결 끊김 시 "연결이 잠깐 끊겼어요. 곧 다시 연결될 거예요." 표시
- [ ] 재연결 후 최신 상태 자동 동기화

### 3.5 무한 캔버스 (줌 & 패닝)

| 항목 | 스펙 |
|---|---|
| 패닝 | 마우스 드래그 (빈 공간) / 트랙패드 2핑거 스크롤 |
| 줌 | 마우스 휠 / 트랙패드 핀치 / 줌 컨트롤 UI |
| 줌 범위 | 10% ~ 400% |
| 미니맵 | 우측 하단 미니맵 (선택, MVP에서는 생략 가능) |
| 홈 복귀 | "Fit to content" 버튼 |

**수용 기준:**
- [ ] 휠/핀치로 줌 인/아웃, 부드러운 애니메이션
- [ ] 빈 공간 드래그로 캔버스 패닝
- [ ] 줌 레벨 UI에 퍼센트 표시
- [ ] 캔버스 이동이 부드럽고 60fps 유지

## 4. 사용자 플로우 상세

### 4.1 신규 캔버스 생성 플로우
```
홈(folio.app) 접속
  → "새 캔버스 만들기" 클릭
  → 서버: 캔버스 생성 + canvasId 발급
  → 리다이렉트: folio.app/{canvasId}
  → 캔버스 화면 진입 (빈 캔버스)
  → 공유 모달 자동 표시 "이 링크를 공유하면 같이 시작할 수 있어요."
  → 링크 복사 / 닫기
```

### 4.2 초대받은 사용자 입장 플로우
```
공유 링크(folio.app/{canvasId}) 클릭
  → 닉네임 입력 모달 (선택, "건너뛰기" 가능)
  → 캔버스 화면 진입
  → 기존 콘텐츠 로드 + 실시간 연결
  → 접속자 목록에 추가
```

### 4.3 콘텐츠 추가 플로우
```
캔버스 화면
  → 하단 툴바: [텍스트] [이미지] [손바닥(패닝)]
  → 텍스트: 클릭 → 캔버스 클릭 위치에 스티커 생성 → 인라인 편집
  → 이미지: 클릭 → 파일 선택 → 업로드 → 캔버스에 배치
  → 또는: 파일을 캔버스에 드래그 앤 드롭
```

## 5. 비기능 요구사항

### 성능
| 지표 | 목표 |
|---|---|
| 캔버스 첫 렌더 (FCP) | 3초 이내 |
| 실시간 동기화 지연 | 500ms 이내 |
| 이미지 업로드 응답 | 2초 이내 (5MB 기준) |
| 캔버스 조작 (줌/패닝) | 60fps |
| Lighthouse 성능 점수 | 80+ |

### 보안
- 캔버스 URL은 nanoid(8자리)로 추측 불가
- 이미지 업로드 시 파일 타입 검증 (서버 사이드)
- XSS 방지: 텍스트 입력 sanitize
- Rate Limiting: 캔버스 생성 IP당 10개/시간
- 파일 업로드: 5MB 제한, 서버에서 재검증

### 접근성
- 모든 인터랙티브 요소 aria-label 적용
- 키보드 네비게이션: Tab, Enter, Delete, Escape
- 색상 대비율 WCAG AA 이상
- 스크린 리더 호환 (주요 액션)

### 브라우저 지원
- Chrome 90+, Safari 15+, Firefox 90+, Edge 90+
- 모바일: iOS Safari 15+, Chrome Android 90+

## 6. 데이터 모델

### Canvas
```
Canvas {
  id: string (nanoid 8자리)
  created_at: timestamp
  expires_at: timestamp (created_at + 7일)
  title: string (optional, 기본값: "제목 없는 캔버스")
}
```

### Element
```
Element {
  id: string (uuid)
  canvas_id: string (FK → Canvas.id)
  type: enum ('sticky' | 'image')
  position_x: float
  position_y: float
  width: float
  height: float
  content: jsonb {
    // sticky: { text: string, color: string }
    // image: { url: string, original_name: string }
  }
  z_index: integer
  created_by: string (session_id)
  created_at: timestamp
  updated_at: timestamp
}
```

### Session
```
Session {
  id: string (uuid)
  canvas_id: string (FK → Canvas.id)
  nickname: string (optional)
  color: string (아바타 컬러)
  connected_at: timestamp
  last_seen_at: timestamp
}
```

## 7. API 엔드포인트 목록

| Method | Endpoint | 설명 |
|---|---|---|
| POST | `/api/canvas` | 캔버스 생성 |
| GET | `/api/canvas/:id` | 캔버스 메타데이터 조회 |
| PATCH | `/api/canvas/:id` | 캔버스 제목 수정 |
| DELETE | `/api/canvas/:id` | 캔버스 삭제 |
| POST | `/api/upload` | 이미지 업로드 (→ Supabase Storage) |
| POST | `/api/liveblocks-auth` | Liveblocks 인증 토큰 발급 |
| Liveblocks | Room: `canvas-{id}` | 엘리먼트 CRUD + 실시간 동기화 |
| Cron | 매일 00:00 UTC | 만료 캔버스 정리 |

## 9. 리스크 및 제약사항

| 리스크 | 영향 | 대응 |
|---|---|---|
| Liveblocks 무료 플랜 제한 | 동시 접속 제한 | 무료 티어 확인, 초과 시 PartyKit 대안 |
| 캔버스 엘리먼트 과다 | 성능 저하 | 엘리먼트 100개 소프트 리밋, 경고 표시 |
| 이미지 스토리지 비용 | 비용 증가 | 7일 후 자동 삭제, 이미지 압축 |
| 악의적 콘텐츠 업로드 | 법적/평판 리스크 | 신고 버튼, 수동 검토 (MVP 이후) |
| 모바일 캔버스 조작성 | UX 저하 | 터치 최적화, 주요 기능만 모바일 지원 |
